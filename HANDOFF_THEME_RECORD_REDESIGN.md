# 主题释放数据库复杂主题改造交接

更新时间：2026-06-29

## 当前目标

在当前 PWA 基础上改造“主题释放数据库”中几个双列/分层主题的记录结构和手机端编辑 UI：

- 成功
- 喜欢与不喜欢
- 化解卡住
- 目标表格 + 目标行动清单合并
- “开始释放”里的主题释放流程也要同步改造，练习过程产生的数据要写入同一套主题释放记录结构。

页面大结构保持不变：

1. 上方是“添加释放记录”
2. 下方是“已有释放记录”
3. 也可以从已有释放记录里继续编辑/查看/新建

特别说明：

- `目标表格` 和 `目标行动清单` 后续应合并成一个目标记录结构，不再作为两个割裂主题处理。
- 数据库页和“开始释放”页必须使用同一套字段语义，避免一边是结构化记录、一边只保存扁平 `items`。

## 术语约定

统一使用“感觉好了吗”，不要写“感受好了吗”。

层级含义：

- `释放了吗`：属于每一张感受/想要卡片。
- `感觉好了吗`：属于更高一层分区，但视觉顺序放在该分区下所有感受/想要卡片之后。
- `收获`：属于整条主题释放记录，每条记录只有一个收获字段。

## 数据层级设计

### 成功

成功主题结构：

```text
主题

如果成功
  感受卡片 1
    情绪/感受
    释放了吗
  感受卡片 2
    情绪/感受
    释放了吗
  + 添加感受
  感觉好了吗

如果失败
  感受卡片 1
    情绪/感受
    释放了吗
  + 添加感受
  感觉好了吗

收获
```

要点：

- “如果成功”下面可以有多个感受卡片。
- “如果失败”下面可以有多个感受卡片。
- 每个感受卡片都有自己的 `释放了吗`。
- “如果成功”整个分区只有一个 `感觉好了吗`。
- “如果失败”整个分区只有一个 `感觉好了吗`。
- 整条记录只有一个 `收获`。

### 喜欢与不喜欢

喜欢与不喜欢主题结构：

```text
主题

我喜欢的方面
  方面分区 A
    方面内容
    感受卡片 1
      情绪/感受
      释放了吗
    感受卡片 2
      情绪/感受
      释放了吗
    + 添加感受
    感觉好了吗
  + 添加方面

我不喜欢的方面
  方面分区 A
    方面内容
    感受卡片 1
      情绪/感受
      释放了吗
    + 添加感受
    感觉好了吗
  + 添加方面

收获
```

要点：

- “喜欢的方面”下面可以有多个方面分区。
- “不喜欢的方面”下面可以有多个方面分区。
- 每个方面分区下面可以有若干感受卡片。
- 每个感受卡片都有自己的 `释放了吗`。
- 每个方面分区只有一个 `感觉好了吗`，并放在该方面下所有感受卡片之后。
- 整条记录只有一个 `收获`。

### 化解卡住

化解卡住主题结构：

```text
主题

对我有什么好处
  好处分区 A
    好处内容
    想要卡片 1
      我对此有什么想要？
      释放了吗
    想要卡片 2
      我对此有什么想要？
      释放了吗
    + 添加想要
    感觉好了吗
  + 添加好处

对我有什么坏处
  坏处分区 A
    坏处内容
    想要卡片 1
      我对此有什么想要？
      释放了吗
    + 添加想要
    感觉好了吗
  + 添加坏处

收获
```

要点：

- “好处”下面可以有多个好处分区。
- “坏处”下面可以有多个坏处分区。
- 每个好处/坏处分区下面可以有若干想要卡片。
- 每个想要卡片都有自己的 `释放了吗`。
- 每个好处/坏处分区只有一个 `感觉好了吗`，并放在该分区下所有想要卡片之后。
- 整条记录只有一个 `收获`。

### 目标表格 + 目标行动清单

目标表格和目标行动清单需要合并成同一条目标记录。结构：

```text
目标

我对此有什么感受
  感受卡片 1
    情绪/感受
    释放了吗
  感受卡片 2
    情绪/感受
    释放了吗
  + 添加感受
  感觉好了吗

为了达成目标要做的事
  行动 1
    行动内容
    感受卡片 1
      情绪/感受
      释放了吗
    感受卡片 2
      情绪/感受
      释放了吗
    + 添加感受
    感觉好了吗

  行动 2
    行动内容
    感受卡片 1
      情绪/感受
      释放了吗
    + 添加感受
    感觉好了吗
  + 添加行动

收获
```

要点：

- 一条目标记录包含目标本身、目标感受、行动清单、行动感受和收获。
- 目标本身下面可以有多个感受卡片。
- 每个目标感受卡片都有自己的 `释放了吗`。
- 目标整体只有一个 `感觉好了吗`，并放在目标感受卡片之后。
- 每个行动下面可以有多个感受卡片。
- 每个行动感受卡片都有自己的 `释放了吗`。
- 每个行动只有一个 `感觉好了吗`，并放在该行动下所有感受卡片之后。
- 整条目标记录只有一个 `收获`。
- 后续 UI 和数据上应避免让 `目标表格` 与 `目标行动清单` 成为两条互不关联的记录。

## 开始释放流程与落库

“开始释放”中的主题释放不应只创建一条简单记录，而要根据主题类型把每轮练习同步写入主题释放数据库中的对应结构。

共同规则：

- 每次进入一个新的感受/想要，都生成或定位一张感受/想要卡片。
- 每张感受/想要卡片记录自己的 `释放了吗`。
- 当 `释放了吗 = 否` 时，回到当前感受/想要，继续循环释放问题。
- 当 `感觉好了吗 = 否` 时，询问“当前是什么感受/想要”，并从这个新内容继续循环释放问题。
- 感受类问题需要复用情绪参考区；想要类问题不一定使用情绪参考区，除非后续明确要做想要参考。
- `感觉好了吗` 写在当前分区/方面/行动层级，不写在单张感受卡片上。
- `收获` 在用户选择“不想继续释放”后弹窗填写，写入整条记录的 `收获` 字段。

### 成功：开始释放流程

```text
填写主题

选择问题
  如果成功，我对此有什么感受？
  如果失败，我对此有什么感受？

填写感受
  下方显示情绪参考区

释放循环
  可以感受到它吗？
  可以放它离开吗？
  愿意放它走吗？
  什么时候放它离开呢？
  释放了吗？
    是：写入当前感受卡片 released = true，进入“感觉好吗？”
    否：写入当前感受卡片 released = false，回到当前感受继续循环
  感觉好吗？
    是：写入当前分区 feelsGood = true，进入“还想继续释放吗？”
    否：写入当前分区 feelsGood = false，询问当前是什么感受，显示情绪参考区，再继续循环

还想继续释放吗？
  想：继续选择“如果成功/如果失败”，新建下一张感受卡片
  不想：弹出收获窗口
```

落库对应：

- `如果成功` 对应成功记录中的 success 分区。
- `如果失败` 对应成功记录中的 failure 分区。
- 每次填写的感受进入所选分区的感受卡片数组。
- `释放了吗` 写入该感受卡片。
- `感觉好吗` 写入所选分区。
- `收获` 写入整条成功记录。

```text
  我喜欢该主题的哪些方面？
  我不喜欢该主题的哪些方面？

填写方面

  愿意放它走吗？
  什么时候放它离开呢？
  释放了吗？

    否：写入当前感受卡片 released = false，回到当前感受继续循环
  感觉好吗？
    是：写入当前方面分区 feelsGood = true，进入“还想继续释放吗？”
    否：写入当前方面分区 feelsGood = false，询问当前是什么感受，显示情绪参考区，再继续循环

还想继续释放吗？
  想：继续选择“喜欢/不喜欢”，填写新的方面并继续循环

```

落库对应：

- `我喜欢该主题的哪些方面` 对应 likes 分区。

### 其他带归属列的主题

对于其他有归属列的主题释放，例如“压抑或表达”等，不需要改成复杂分层结构，但开始释放流程要在原本第一步和第二步之间加入归属列下拉选择。

通用顺序：

```text
填写主题
选择归属列
填写感受/想要
进入原有释放循环
同步保存到释放记录
```

落库对应：

- `column` 保存用户选择的归属列。
- 感受/想要继续进入原有 `items` 或后续兼容结构。
- `释放了吗` 仍属于单条感受/想要。
- `感觉好了吗` 仍属于整条简单记录。
- `收获` 仍属于整条记录。

## 手机端 UI 排布

### 主题详情页整体

保持当前主题释放数据库页的上下结构：

```text
主题说明

添加释放记录
  新建一条该主题记录

已有释放记录
  记录卡片
  记录卡片
```

### 添加释放记录区

对复杂主题使用纵向卡片结构，不使用横向表格。

建议顺序：

```text
添加释放记录

主题
[ 输入主题 ]

分区 1
  item / 子分区 / 感受卡片
  ...
  感觉好了吗

分区 2
  item / 子分区 / 感受卡片
  ...
  感觉好了吗

收获
[ 输入收获 ]

[ 保存记录 ]
```

### 已有释放记录区

已有记录卡片默认摘要展示，避免手机页面过长。

成功记录摘要示例：

```text
我的目标主题
如果成功：3 个感受 · 感觉好了
如果失败：2 个感受 · 还不好
收获：已填写
```

喜欢与不喜欢摘要示例：

```text
某个人
喜欢：2 个方面 · 5 个感受
不喜欢：3 个方面 · 4 个感受
收获：未填写
```

化解卡住摘要示例：

```text
卡住的事情
好处：3 条 · 4 个想要
坏处：2 条 · 3 个想要
收获：已填写
```

点开已有记录后进入完整编辑结构。

## 后续实现建议

1. 先补数据结构适配，不要直接破坏旧 `TopicRecord.items`。
2. 可以新增复杂主题专用字段，例如 `structuredSections`，同时保留旧字段用于旧记录兼容。
3. 只对以下 topic id 使用新结构：
   - `success`
   - `likes-dislikes`
   - `stuckness`
   - `goal-feelings`
   - `goal-actions`
4. 其他主题继续使用现有简单记录结构。
5. 添加迁移/兼容函数，把旧记录摘要显示出来，避免已有数据丢失。
6. 修改导出逻辑，让复杂结构能导出完整层级。
7. 修改“开始释放未完成项”时的取值逻辑，能够从复杂结构中定位未释放的感受/想要卡片。
8. 对 `goal-feelings` 和 `goal-actions` 做合并迁移策略：旧目标感受记录和旧行动记录需要能合并展示到同一条目标记录里，或者至少保留兼容入口。
9. 修改 `createSession` 的入参或新增上下文字段，让一次释放会话知道自己属于哪个结构分区，例如成功/失败、喜欢/不喜欢、好处/坏处、目标行动。
10. 修改释放轮次结束后的回写逻辑。当前代码只会按 `session.currentFeeling` 更新 `record.items`，复杂结构需要按结构路径更新对应感受/想要卡片、所在分区的 `感觉好了吗`，以及整条记录的 `收获`。
11. 修改“还想继续释放吗”的处理：选择继续时不要新建割裂记录，而是在当前主题记录下追加新的分区、方面、行动或感受/想要卡片。
12. 保留旧记录兼容：旧的扁平 `items` 记录仍然可读、可导出、可继续释放。
